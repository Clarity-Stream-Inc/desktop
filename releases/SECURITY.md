# Security Policy

## Reporting a vulnerability

If you discover a security issue in the Clarity Stream desktop app or any Clarity Stream service, please report it privately.

**Email:** security@claritystream.io
**PGP key:** https://claritystream.io/.well-known/security.txt

Please include:
- A description of the issue and its potential impact
- Steps to reproduce (proof-of-concept where possible)
- Affected version(s) and platform (macOS / Windows)
- Your contact details

We will acknowledge within **2 business days** and aim to provide a remediation timeline within **10 business days**.

## Scope

In scope:
- The Clarity Stream desktop app (this repo's release artifacts)
- The web dashboard at `claritystream.io`
- Clarity Stream backend / Edge Functions

Out of scope:
- Social engineering, physical attacks
- Denial-of-service via volumetric traffic
- Vulnerabilities in third-party dependencies already disclosed upstream

## Safe harbor

We will not pursue legal action against researchers acting in good faith and within this policy.

## Supported versions

Only the **latest released version** of the desktop app receives security updates. Auto-update is enabled by default — please keep it on.
